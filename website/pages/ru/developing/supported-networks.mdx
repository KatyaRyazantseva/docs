<p space-before="0">
  Хостинговый сервис опирается на стабильность и надежность базовых технологий, а именно предоставляемых конечных точек JSON RPC.
</p>

<p space-before="0">
  Ropsten, Rinkeby и Kovan устарели. Подробнее читайте в <a href="https://blog.ethereum.org/2022/06/21/testnet-deprecation">блоге Ethereum Foundation</a>. По состоянию на 25 февраля 2023 г. Ropsten, Rinkeby и Kovan больше не поддерживаются Хостинговым сервисом. После слияния Goerli будет поддерживаться разработчиками клиентов, а также Хостинговым сервисом. Разработчикам, которые в настоящее время используют Ropsten, Rinkeby или Kovan в качестве промежуточной среды/среды тестирования, рекомендуется перейти на Goerli.
</p>

<p space-before="0">
  Субграфы, индексирующие Gnosis Chain, теперь могут быть развернуты с помощью сетевого идентификатора <code>gnosis</code>. <code>xdai</code> по-прежнему поддерживается для существующих субграфов хостинговых сервисов.
</p>

Полный список функций, поддерживаемых в децентрализованной сети, можно найти на [этой странице](https://github.com/graphprotocol/indexer/blob/main/docs/feature-support-matrix.md).

Субграфы, работающие на основе субпотоков, индексирующие `mainnet` Ethereum, поддерживаются в Subgraph Studio и децентрализованной сети.

## Graph Node

Если предпочитаемая Вами сеть не поддерживается в децентрализованной сети The Graph, Вы можете запустить свой собственный [Graph Node](https://github.com/graphprotocol/graph-node) для индексирования любой сети, совместимой с EVM. Убедитесь, что используемая Вами [версия](https://github.com/graphprotocol/graph-node/releases) поддерживает сеть, и у Вас есть необходимая конфигурация.

Graph Node также может индексировать другие протоколы посредством интеграции с Firehose. Интеграция Firehose была создана для сетей на базе NEAR, Arweave и Cosmos.
